[Idea Canvas Web](https://idea-canvas-web.ysmyapp.workers.dev/) を公開した。これは、アイデアや検討事項をカードとして並べ、ブラウザ上で動的に整理できるWebアプリだ。

単にテキストを入力して一覧表示するだけではなく、カードを対話的に動かしながら、考えを整理することを目的にしている。アイデア出し、タスクの分類、検討中の論点整理など、まだ形になりきっていない情報を扱うための小さなキャンバスとして作った。

## 作ったもの

Idea Canvas Web は、カード型のUIでアイデアを扱うアプリケーションである。ユーザーはカードを追加し、内容を書き、並び替えながら全体の構造を調整できる。

このアプリで見せたかったのは、Next.js と React を使うことで、静的なWebページだけでなく、ユーザー操作に反応して状態が変わるインタラクティブなアプリケーションも作れるという点だ。

## Next.js と React で作った理由

Next.js はページ構成やデプロイの見通しを立てやすく、React はカードの追加、編集、並び替えのような状態を持つUIを作りやすい。

今回のようなアプリでは、ユーザーの操作に応じて画面上のカードの並びや内容が変わる。そのため、React の state を使って画面状態を管理し、コンポーネント単位でUIを組み立てる構成が自然だった。

また、Cloudflare Workers に公開することで、ポートフォリオとしてすぐに触れる形にできた。コードだけでなく、実際にブラウザで操作できるものとして見せられる点が重要だと考えた。

## Supabase ではなくブラウザ保存にした理由

最初は Supabase を使ってデータベースに保存する構成も考えた。ユーザーごとにデータを保存したり、ログイン機能を付けたりするなら、Supabase のようなBaaSはかなり便利だ。

ただし、今回の目的は長期的にポートフォリオとして公開し続けることだった。無料枠の条件変更、プロジェクト停止、利用期限、認証やDB管理の手間が増えると、見せたいものに対して構成が重くなる。

そのため、今回はブラウザ側に保存する形を選んだ。これなら、バックエンドやDBを用意しなくてもアプリとして成立する。見る人がすぐに触れて、カードの追加や並び替えを試せる。ポートフォリオ用途では、このシンプルさにメリットがある。

もちろん、この判断にはトレードオフもある。

| 保存方法 | メリット | 注意点 |
| --- | --- | --- |
| Supabase | 複数端末同期、ログイン、共有、永続DBに向いている | 認証、DB設計、無料枠、運用管理を考える必要がある |
| ブラウザ保存 | 構成が軽く、ポートフォリオとして長期公開しやすい | 端末やブラウザをまたいだ同期には向かない |

今回のアプリでは、まず「触れるポートフォリオ」として安定して公開することを優先した。そのため、ブラウザ保存の方が目的に合っていると判断した。

## ポートフォリオとして見せたい点

このアプリでは、次のような点を見せたい。

- Next.js と React で動的なWebアプリを作れること
- カード型UIのような対話的な画面を構築できること
- ユーザー操作に応じて状態を更新できること
- Cloudflare Workers に公開して、実際に触れる形でデプロイできること
- DBを使う場合と使わない場合の設計判断ができること

ポートフォリオでは、使った技術名だけでなく、なぜその構成にしたのかを説明できることが大事だと思う。今回は、Supabase を使う案も考えたうえで、長期公開と運用の軽さを優先してブラウザ保存にした。

## 今後の改善案

今後改善するなら、次のような方向が考えられる。

- カードの色やカテゴリ分け
- キーワード検索
- エクスポート / インポート
- JSON や Markdown 形式での保存
- 複数キャンバスの管理
- 必要になった場合の Supabase 連携

特に、エクスポート機能があると、ブラウザ保存の弱点を少し補える。ユーザーが自分のデータをファイルとして持ち出せるようにすれば、DBなしでも実用性が上がる。

## まとめ

Idea Canvas Web は、Next.js と React を使って、カードを動的に並び替えながらアイデアを整理するWebアプリとして作成した。

今回は、長期的にポートフォリオとして見せることを重視し、Supabase ではなくブラウザ保存を採用した。DBや認証を使わなくても、目的に合った範囲であれば十分に実用的なWebアプリを作れる。

技術的には小さなアプリだが、React による状態管理、対話的なUI、Cloudflare Workers へのデプロイ、保存方式の設計判断をまとめて見せられる制作物になった。
